# Adding Mitos to a project

Mitos is **per-project**: each project gets its own decision graph and its own
Qdrant collection, sharing one Qdrant instance. Setting it up for a new project
takes a couple of minutes. Run **`mitos status`** at any time to see exactly
what's done and what's left (it exits `0` when ready, `1` otherwise).

> **For LLM agents:** run `mitos status` first. If it isn't `READY`, follow the
> steps below, then re-run `mitos status` to confirm. When done, do **step 6** so
> the next agent in this project inherits the setup.

---

## Prerequisites (once per machine)

- **Mitos installed** — `pip install -e .` from the [mitos repo](https://github.com/dovahkiin-v/mitos), or `pip install mitos-adr`.
- **Docker** — for Mitos's Qdrant.
- **A Google Gemini API key** — <https://aistudio.google.com/app/apikey>. (An Anthropic key is only needed for `mitos import --llm-extract`.)

### Start Mitos's Qdrant (once per machine, shared by all projects)
```bash
docker compose up -d        # from the mitos repo → mitos-qdrant on :7333
```
Mitos uses its **own** Qdrant on `:7333` — *not* the standard `:6333` — so it
never lands in another Qdrant you run for other work. It fails safe: if Qdrant
isn't up, `mitos record` still commits to the graph and queues embeddings; only
semantic search pauses until you start it.

---

## Per-project setup

### 1. Initialize the workspace
From the project root:
```bash
mitos init
```
This creates `.mitos/` (graph + config + skill), `decisions.md`, `format-spec.md`,
and scaffolds a **gitignored `.env`** with empty key slots. The project gets its
own collection, `mitos-<project>`.

### 2. Add your key
Put your Gemini key in the project's `.env` (it's gitignored — never commit it):
```
GEMINI_API_KEY=your-key-here
```
`mitos` loads `.env` automatically. One key covers embeddings *and* synthesis.

### 3. Wire the MCP server into your agent
So your agent can `surface_decisions` / `query_decisions` / `record_decision`,
point it at `mitos serve` (a stdio MCP server) with the **project as its cwd**.

**Claude Code** — add a project-scoped `.mcp.json` at the project root:
```json
{
  "mcpServers": {
    "mitos": { "command": "mitos", "args": ["serve"] }
  }
}
```
**Other harnesses** (Cursor, Gemini CLI, custom): register `mitos serve` as an
MCP server, run with the project directory as the working directory. The
`.mitos/skill.md` generated by `init` is the integration prompt — load it (or
reference it from your agent instructions) so the agent uses Mitos correctly.

### 4. Verify
```bash
mitos status        # expect: READY ✓
```

### 5. (Optional) Backfill existing decisions
If the project already has a prose ADR log, import it once:
```bash
mitos import --from prose --llm-extract path/to/DECISIONS.md   # needs ANTHROPIC_API_KEY
```

### 6. Tell the next agent — point your project's agent files at this guide
So **any** agent that later opens this project knows Mitos is here and how to set
it up, add a short pointer to whichever agent-instruction files the project uses
(`AGENTS.md`, `CLAUDE.md`, `GEMINI.md`, `.cursorrules`, …):

```markdown
## Architectural Decisions — Mitos (per-project setup)
This project uses **mitos** for architectural decision memory. It is a
**per-project** setup. Run `mitos status` to check it; if not ready, follow the
mitos setup guide: https://github.com/dovahkiin-v/mitos/blob/main/SETUP.md
Record decisions with `record_decision`; check precedents with `surface_decisions` / `query_decisions`.
```

That line is what makes Mitos-awareness travel with the project across agents and
sessions.
